HTML Entities

Learn to display special characters, symbols, and reserved HTML characters using HTML entities for proper text rendering.

Understanding HTML Entities

HTML entities are special codes used to display characters that have specific meanings in HTML or characters that are not readily available on standard keyboards. They ensure proper rendering of text content without conflicts with HTML markup syntax.

Entities are essential for displaying characters like less-than (<) and greater-than (>) signs, copyright symbols, accented characters, mathematical symbols, and special typography characters in web pages.

🔤 Key Concept

HTML entities allow you to display special characters that would otherwise be interpreted as HTML code or are not available on standard keyboards.

Entity Syntax

HTML entities can be represented using named entities or numeric character references: 

<!-- Named entities (human-readable) -->
<p>This &lt;tag&gt; will be displayed as text.</p>
<!-- Displays: This <tag> will be displayed as text. -->

<!-- Numeric character references (decimal) -->
<p>Copyright &#169; 2025 My Company</p>
<!-- Displays: Copyright © 2025 My Company -->

<!-- Hexadecimal character references -->
<p>Arrow: &#x2192;</p>
<!-- Displays: Arrow: → -->

<!-- Mixed usage in content -->
<p>
    The equation is: x &lt; y &amp;&amp; y &gt; z
    &ndash; that&rsquo;s &ldquo;mathematics&rdquo;!
</p>
<!-- Displays: The equation is: x < y && y > z – that's "mathematics"! -->
Entity Syntax Patterns:
  • Named: &name; (e.g., &copy;)
  • Decimal: &#number; (e.g., &#169;)
  • Hexadecimal: &#xnumber; (e.g., &#xA9;)
  • All entities start with & and end with ;

Reserved HTML Characters

<!-- Essential reserved characters -->
<p>
    To display HTML tags as text:
    &lt;div class=&quot;example&quot;&gt;Content&lt;/div&gt;
</p>
<!-- Displays: To display HTML tags as text: <div class="example">Content</div> -->

<!-- Ampersand in URLs and text -->
<p>
    Visit: example.com?name=John&amp;age=25&amp;city=NYC
</p>
<!-- Displays: Visit: example.com?name=John&age=25&city=NYC -->

<!-- Quotes in attributes and text -->
<p title="She said &quot;Hello&quot; to me">
    He replied: &quot;Nice to meet you!&quot;
</p>

<!-- Non-breaking space -->
<p>
    First Name:&nbsp;&nbsp;&nbsp;John&nbsp;Doe
</p>
<!-- Creates fixed spaces that won't break -->

Reserved Characters

  • &lt; → < (less than)
  • &gt; → > (greater than)
  • &amp; → & (ampersand)
  • &quot; → " (double quote)
  • &apos; → ' (apostrophe)

Special Spaces

  • &nbsp; → Non-breaking space
  • &ensp; → En space (width of 'n')
  • &emsp; → Em space (width of 'm')
  • &thinsp; → Thin space
  • &#8203; → Zero-width space

Common Symbols and Characters

<!-- Copyright and trademark symbols -->
<p>
    &copy; 2025 My Company&trade;. All rights reserved&reg;.
</p>
<!-- Displays: © 2025 My Company™. All rights reserved®. -->

<!-- Currency symbols -->
<p>
    Prices: &dollar;19.99, &euro;15.50, &pound;12.75, &yen;2000
</p>
<!-- Displays: Prices: $19.99, €15.50, £12.75, ¥2000 -->

<!-- Mathematical symbols -->
<p>
    Math: x &plusmn; y &times; z &divide; 2 = &infin;
    &sum; &int; &radic;x &le; &ge; &ne; &equiv;
</p>
<!-- Displays: Math: x ± y × z ÷ 2 = ∞ ∑ ∫ √x ≤ ≥ ≠ ≡ -->

<!-- Arrows and geometric shapes -->
<p>
    Directions: &uarr; &darr; &larr; &rarr; &harr; &crarr;
    Shapes: &#9632; &#9633; &#9650; &#9660; &#9679; &#9675;
</p>
<!-- Displays: Directions: ↑ ↓ ← → ↔ ↵ Shapes: ■ □ ▲ ▼ ● ○ -->

<!-- Punctuation and typography -->
<p>
    Typography: &lsquo;single quotes&rsquo; and &ldquo;double quotes&rdquo;
    &ndash; en dash, &mdash; em dash, &hellip; ellipsis
</p>
<!-- Displays: Typography: 'single quotes' and "double quotes" – en dash, — em dash, … ellipsis -->

Entity Reference Tables

Basic Symbols

  • &copy; → © (copyright)
  • &reg; → ® (registered)
  • &trade; → ™ (trademark)
  • &deg; → ° (degree)
  • &sect; → § (section)
  • &para; → ¶ (paragraph)

Quotation Marks

  • &lsquo; → ' (left single quote)
  • &rsquo; → ' (right single quote)
  • &ldquo; → " (left double quote)
  • &rdquo; → " (right double quote)
  • &laquo; → « (left guillemet)
  • &raquo; → » (right guillemet)

Dashes

  • &ndash; → – (en dash)
  • &mdash; → — (em dash)
  • &minus; → − (minus sign)
  • &#8208; → ‐ (hyphen)
  • &#8209; → ‑ (non-breaking hyphen)

Special Characters

  • &hellip; → … (ellipsis)
  • &middot; → · (middle dot)
  • &bull; → • (bullet)
  • &#8203; → Zero-width space
  • &#8288; → Word joiner

Best Practices

✅ Do

  • Always escape reserved HTML characters
  • Use named entities for common characters
  • Use UTF-8 encoding when possible
  • Use non-breaking spaces for fixed spacing
  • Include proper character set declarations
  • Test entity rendering across browsers

❌ Avoid

  • Using entities for characters available in UTF-8
  • Forgetting to escape ampersands in URLs
  • Using incorrect entity syntax
  • Over-using entities when direct Unicode works
  • Mixing character encodings
  • Using deprecated entity names
